# 双角色推演方法论

## 概述
通过两个独立子Agent分别扮演角色A和角色B，主Agent负责世界观搭建、回合推进、结果推演和状态管理。适用于双人对抗、合作或平行发展的推演场景。

---

## 一、初始化阶段

### 1.1 创建世界
- 使用 `world_create` 创建独立世界（worldId用连字符，不用下划线）
- 使用 `world_switch` 切换到新世界
- 在根节点 n0000 写入世界观背景（`write_element ref="worldview:背景设定"`）

### 1.2 创建子Agent配置
- 使用 `create_sub_agent_config` 为每个角色创建独立Agent
- agent_id 用角色名拼音（如 zhangxiaoming、lihua）
- system_prompt 需包含：背景、性格、技能、目标、行为准则
- 行为准则关键：第一人称、时代认知局限、具体行动输出
- temperature 建议 0.7（增加角色自主性）
- tool_filter 设为 read_only

### 1.3 创建第一回合节点
- 使用 `node_create` 创建 n0001（或后续编号）
- worldTime 格式：`1983年3月，地点，季节/氛围`
- 写入角色初始状态到 `characters:角色名`
- 写入开局叙事到 `narrative:开局叙事`

---

## 二、回合推进循环

### 2.1 创建新节点
- 使用 `node_create` 创建下一回合节点
- 回合间隔通常1-2个月，根据剧情节奏调整
- 关键节点（考核、冲突、转折）可缩短间隔

### 2.2 派发子Agent生成行动
- 使用 `dispatch_sub_agent` 分别派发两个角色Agent
- prompt 需包含：
  - 当前时间和地点
  - 角色当前状态（从上一回合角色状态元素提取）
  - 近期变化（新情报、新机会、新威胁）
  - 要求输出：内心想法 + 具体行动（1-2件）
- 两个 dispatch 可并行调用
- 注意：子Agent输出可能被截断，从已有内容提取行动意图即可

### 2.3 主Agent推演结果
- 基于子Agent的行动意图，结合时代背景和NPC性格推演
- 每件事给出：过程简述、关键对话片段、结果判定（成功/部分成功/失败）、得失分析
- 推演原则：
  - 符合时代背景（如1983年的社会规则、物资条件）
  - NPC有独立利益考量，不会无条件配合
  - 行动有代价（花钱、花时间、欠人情）
  - 允许部分成功——得到一些，失去一些
  - 引入意外因素（竞争对手、政策变化、运气）

### 2.4 写入叙事和更新状态
- 推演结果写入 `narrative:第X回合行动与结果`
- 更新角色状态到 `characters:角色名`（覆盖模式）
- 状态更新要点：积蓄变化、新社会关系、技能进步、新增压力/机会

## 三、关键设计原则

### 3.1 角色差异化
- 两个角色应有不同的出身、性格、路线
- 路线可以互补（经商 vs 体制）或对立（竞争同一资源）
- 初始资源应有差距但不过于悬殊

### 3.2 时代感
- 所有推演必须符合时代背景
- 物价、工资、社会规则要真实
- 角色认知不能超越时代（如1983年的人不知道未来）

### 3.3 代价与权衡
- 每次行动都有代价
- 角色必须在多个选项间做取舍
- 没有完美选择，只有权衡

### 3.4 渐进式交集
- 初期两人各自发展，不强行交集
- 通过地理位置、共同事件、第三方人物自然产生联系
- 交集时机：资源竞争、意外相遇、共同危机

## 四、常见问题

| 问题 | 解决方案 |
|------|---------|
| node_create 返回成功但节点不在链上 | 用 node_list 确认，必要时重新创建 |
| 子Agent输出被截断 | 从已有内容提取行动意图，不需要完整输出 |
| sim 子代理超时 | 主Agent直接推演，不依赖 sim |
| 角色状态丢失 | 每回合结束时更新 characters 检查点 |
| 两个角色长期无交集 | 在 prompt 中埋入交集线索（如同一条街、共同认识的人）
